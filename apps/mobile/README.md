# ALIS Mobile

Android mobile app for ALIS. It combines the data cockpit, Nutrition and AI
Coach with native Health Connect background synchronization.

## Configuration

Default development API:

```text
http://localhost:8010
```

On a physical Android phone, set the API URL to the LAN or Tailscale address of
the Docker host from the app settings. The pairing code is generated by
`../../scripts/install.sh`.

Optional build-time variables:

```bash
EXPO_PUBLIC_HEALTH_API_URL=http://YOUR_SERVER_IP:8010
EXPO_PUBLIC_PAIRING_CODE=YOUR_PAIRING_CODE
```

The device token is stored locally with `expo-secure-store` and reused by the
dashboard, Nutrition, Coach and Health Connect sync.

## Commands

```bash
npm install
npm test
npm run type-check
npx expo run:android
```

For release builds, provide your own Android keystore. Do not commit keystores
or generated APK/AAB files.
